# Support

Thank you for your interest in the **Cybersecurity Handbook**.

This document explains how to get help, ask questions, report problems, and contribute to the project.

---

# 📚 Before Asking for Help

Please check the following resources first:

- README.md
- CONTRIBUTING.md
- SECURITY.md
- docs/HANDBOOK_ARCHITECTURE.md
- docs/STYLE_GUIDE.md
- Existing GitHub Issues
- Existing Discussions (when enabled)

Many common questions are already answered there.

---

# 💬 General Questions

For general questions about:

- Cybersecurity concepts
- Labs
- Learning roadmap
- Career guidance
- Documentation

Please use **GitHub Discussions** (future) instead of creating an Issue.

---

# 🐞 Bug Reports

Create a GitHub Issue when you find:

- Incorrect technical information
- Broken links
- Formatting issues
- Diagram problems
- Markdown rendering issues
- Repository problems

Include:

- Description
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Suggested improvement

---

# ✨ Feature Requests

Feature requests are welcome.

Examples include:

- New cybersecurity topics
- Additional labs
- More interview questions
- Better diagrams
- New learning paths
- New enterprise scenarios
- Additional certifications
- Automation improvements

Please explain:

- Why the feature is valuable
- Proposed solution
- Expected benefits

---

# 📖 Documentation Improvements

Documentation improvements are encouraged.

Examples:

- Grammar corrections
- Better explanations
- Improved diagrams
- Additional references
- Accessibility improvements

Small contributions are just as valuable as large ones.
